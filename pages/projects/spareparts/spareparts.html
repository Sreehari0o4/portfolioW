Overview

The Spare Parts Management System is a responsive web application developed to streamline the tracking and organization of spare parts used in public transport maintenance. It helps the administration efficiently manage inventory, issue records, and part replacements, reducing manual paperwork and improving accessibility.

Problem Statement

Public transport systems often face delays due to untracked spare parts and inefficient inventory systems. The goal of this project was to create a centralized and digital solution to manage spare parts, enabling faster retrieval, real-time updates, and better accountability.

  • Created an intuitive admin dashboard
  • Set up backend APIs and database schema
  • Implemented forms for part entry, issue tracking, and history logs
  • Ensured mobile responsiveness and user-friendly interactions

Challenges and Solutions

  • Challenge: The existing manual system led to confusion over available stock, delays in maintenance, and unrecorded usage.
    Solution: Implemented dynamic CRUD operations with a real-time database to automatically update stock levels when items were issued or added.
  • Challenge: Staff members unfamiliar with digital tools found it hard to adopt complex UIs.
    Solution: Designed a minimal, intuitive interface with clear icons, tooltips, and categorized navigation to reduce learning time.
  • Challenge: Duplicate entries and mismatched part names created confusion.
    Solution: Added input validation, auto-suggestions, and unique ID generation for each spare part to maintain consistency.

Outcome and Result

  • Improved Efficiency: Reduced time spent on searching, recording, and issuing spare parts by automating the entire flow.
  • Organized Records: All transactions and inventory details were stored in a searchable digital format, ensuring traceability and accountability.
  • The clean and accessible design received positive feedback from testers who had minimal digital experience.
